
# 目标
开发一个叫做ccc的agent，类似codex和claude code 。 需要着重突出代码能力。

# 名称
ccc 的全称为 Cross Code Companion，中文名暂定为“跨模型代码助手”。

Cross 表示跨模型、跨工具、跨协议和跨工作流；Code 表示核心场景是代码理解、修改与验证；Companion 表示产品形态是长期陪伴开发者工作的本地代码智能体。

# 语言
前端用react， pnpm ， 后端用python

# plan模型

第一版需要把 Plan 作为一等数据模型。这里的 Plan 不是模型厂商的“大模型型号”，而是 agent 在执行复杂代码任务前生成的任务计划。

Plan 至少包含目标、范围、步骤、每一步状态、风险点、需要用户确认的操作、验证方式和最终状态。Plan 需要写入 transcript，并可单独保存到 session 目录，方便恢复、审计和后续继续执行。

Plan 模式下默认只允许读取仓库、分析上下文和生成计划，不允许直接写文件或执行有副作用的命令。用户确认后才能进入 execute 模式。

# 运行模式

第一版需要区分以下运行模式：

- `chat`：普通对话和代码问答，可读取上下文，低风险工具可用。
- `plan`：只生成任务计划，不修改文件，不执行有副作用命令。
- `execute`：执行已确认的计划，可按审批策略调用写文件、patch、shell 等工具。
- `review`：代码审查/解释模式，默认只读。
- `serve`：启动本地 HTTP 服务，供 React TUI 风格 Web 前端调用。

CLI-first 形态下，`ccc chat`、`ccc plan`、`ccc run`、`ccc serve` 都通过同一套本地 AgentHost / AgentRuntime 调用核心能力；Web 形态下，React 前端通过 HTTP API 传入 `mode`。
