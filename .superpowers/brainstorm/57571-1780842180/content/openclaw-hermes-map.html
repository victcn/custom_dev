OpenClaw vs Hermes: Long-Lived Agent Learning Map

Current working model from repository docs and source orientation. This is a learning scaffold, not a final conclusion.

OpenClaw: local-first assistant control plane
Gateway daemon
WebSocket RPC, channel adapters, node pairing, health, events
Session lanes
Per-session serialization, queue modes, transcript write locks
Embedded Pi agent runtime
Prompt assembly, model stream, tool execution, lifecycle events
Workspace memory
Markdown memory files, memory-core search, dreaming, compaction flush
Plugins/hooks/nodes
Provider plugins, tool hooks, mobile/macOS nodes, canvas/browser surfaces
Hermes: self-improving agent runtime
AIAgent loop
Python tool loop, provider transports, context compressor, session history
Gateway + platforms
Messaging platform adapters, routing, dynamic session context
MemoryManager
Provider orchestration, prefetch, fenced memory context, sync after turn
Skill system + curator
Skill creation, usage tracking, background review, archive/consolidate/patch
Cron/subagents/research
Scheduled runs, isolated workstreams, benchmarks and trajectory generation

Three Study Layers

1

Architecture principles

2

Source reading guide

3

Minimal reproduction model

Extract the smallest set of mechanisms needed to build a long-lived agent later.
